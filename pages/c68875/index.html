产品经理必备的技能

在材料或者文献管理方面主要包括收集、整理、分析与追踪，目的是获取当前研究趋势。

用认知过程阶段可以分成三个：从无到有、从有到精与从精到用，从无到有是指刚进入一个新领域时的状态，绝大多数从技术、到项目、到产品的IT行业人员，都要通过这个阶段构建自己的文献知识库；从有到精指维护与整理与追踪新文献；从精到用阶段指文献知识库体系直接参与科研过程形成产出的过程。

Zotero 实贱经验

本文是我自己在使用文献管理软件 Zotero 的过程中总结的经验、实用功能的用法以及（自以为的）最佳实践。本文不是初级教程，所以会选择性忽略一些基础的设置和步骤。

分析与吐槽

我最开始使用的文献管理软件其实是 Mendeley。Mendeley 从很多方面来讲都是一款非常优秀的软件，但是它的一些致命缺陷又让我难以忽视。最突出的一个问题就是文献同步，永远是令人无法忍受的龟速。当然我知道这不是 Mendeley 本身的错，但在我挂了科学上网工具也无济于事的情况下，这确实成为了一个无法回避的问题。在决定弃用 Mendeley 之前，也参考过别人的方法把数据都转移到了 OneDrive 上，但这样一来我在 iPad 上也不能继续使用它的客户端了（因为没有了上传到软件服务器的同步数据），这又让人很难受。其它难以接受的问题还包括了每次打开软件时的自动 retriving 使得文献列表乱序、PDF 编辑内容只能在软件内查看等等。种种不舒适的体验最终让我下定决心改弦更张。而为什么选择 Zotero 呢？因为 Zotero 是除 Mendeley 以外 Researcher（一款文献订阅 APP）唯一支持链接的文献管理软件。

下面进入正题。

1 软件配置

Zotero 的同步功能包括数据同步（即各种元数据 [元数据：文献标题、作者、类型、年份等]、笔记等）和文件同步（即 PDF 等附件）（数据同步和文件同步的区别）。数据同步无限制，而人们常说免费账户中的 300MB 存储空间是针对文件同步而言的。我的文件同步方案是通过 WebDAV 同步，具体步骤网上已经有很多教程了（如这一篇），这里直接贴一张配置图，不再赘述。

自定义数据存储位置

Zotero 的数据文件默认是在 C 盘保存的，虽然 PDF 文件都转移走了，长期积累下去也要占用不小的空间。好在 Zotero 支持很方便地修改数据的存储位置，我选择把它迁移到 D 盘的软件安装目录下。更激进一点的话，还可以像前面对待 Mendeley 一样，把数据文件也迁移到 OneDrive 里面，不过这样会导致每次打开软件后 OneDrive 的小图标都保持待上传状态（图标会多个小圆圈），看着又有点烦。

笔记视图优化

参考这篇文章，在编辑器高级设置中直接编辑 css 代码来修改笔记的行距与字号：

extensions.zotero.note.css
p {line-height: 1.4; margin-bottom: 1en;} li {line-height: 1.2; margin-bottom: 1en;} blockquote {margin-left: 1en;}

extensions.zotero.note.fontSize
15
2. 若要取消拼写检查，可参考这个答案，在编辑器高级设置中修改：

extensions.spellcheck.inline.max-misspellings
0
不同的文献类型与对应的 icon

这一条属于解释性内容，不是对软件的配置。不同类型的文献会对应不同的 icon，刚开始添加文献的时候我百思不得其解为什么有的图标上面会多出一个多人协作的标志，最后才恍然大悟，原来一个是期刊、一个是会议 :-(

2 插件

插件源

官方插件集合页面：Zotero 官方维护了一个插件列表，大多数已有的插件都被索引到了。

GitHub搜索插件：在 GitHub 上开源的插件，有很大一部分已经在前面的官方索引列表中提到了，但还是有一些漏网之鱼，比如。。

下面简单介绍一下我正在用的一些插件。

插件简介

ZotFile

Zotero 必装插件，自不必多说。提两个小点：

Source Folder for Attaching new Files 这个选项会监控选定的文件夹，在该文件夹下新出现的 PDF 文件就可以通过右键“Attach New File”来添加。一些教程把这个目录设置成了 Zotero 的数据存储位置，但这样其实并不实用（反正被拖入的 PDF 文件也会自动检索元数据，最后生成一个新的数据子目录，ZotFile就检测不到了）。所以我把这个目录设置为了浏览器的下载目录。（这也是 ZotFile 官方推荐的做法）
在 Windows 下勾选子目录选项后若发现实际上并没有建立起与规则对应的子目录，把通配符的正斜杠（/）换成反斜杠（\）即可。我这里是按照自己文库下的分组名称（%c）来建立目录的。（ZotFile 的所有通配符）
ZoteroQuickLook

在 Zotero 中按下空格快速预览文献。众所周知 Zotero 没有内置的 PDF 阅读器，因此该插件算是有限场景下的一种折中吧。该插件在 Windows 下默认还需要搭配一款独立软件 QuickLook 才能工作，但我已经有了相同功能的另一款软件 Seer，读文档后发现也是可以替代的，编辑器高级设置中找到 extensions.zoteroquicklook.customviewcommand 一项并设置为 Seer 的完整路径即可。

Zotero Scholar Citations

玩具性质的插件，在“其它”属性下更新文献的 Google Scholar 引用次数。导入文献的时候这个插件会自动更新，也可以在右键菜单中手动选择更新。

注意：由于调用的是 Google Scholar 引用次数，而 Google Scholar 在国内网络环境下并不能访问，所以使用该插件需要先打开科学上网工具并设置为全局模式。

Markdown Here

原本是一个浏览器插件，但是可以无缝移植到 Zotero 上面。参考这篇教程，手动导入。在笔记页面按照 Markdown 语法记录内容，完成后按下 Ctrl+Alt+M 就可以快速渲染。其实这本质上还是得益于 Zotero 的笔记系统支持 html 格式，就算没有 Markdown Here 也可以手动调整 html 来达到同样的效果，只不过这个插件把这个步骤给自动化了，效率自然也得到了提升。用这个插件，主要是想学习一下更系统的文献笔记。

注意：

Markdown Here 使用 GitHub 风格的 Markdown 语法。
笔记在浮动状态下无法使用 Markdown 转换，必须停靠在程序界面内才能触发。
Markdown 转换后，若想要修改内容并再次渲染（比如在某个地方加一对粗体标记“**”），记得在修改任何内容之前先按下 Ctrl+Alt+M 把笔记转换回纯文本模式，否则你在渲染状态下做的修改会丢失。
如果不幸没有遵照上面一条，发现自己修改的内容丢失了，不要慌，按下 Ctrl+Z 可以撤回操作。
Better BibTeX for Zotero

用于管理和导出 Bib(La)TeX 引文数据，方便 LaTeX 调用。轻度使用直接参考这篇教程的配置即可，想折腾的话在 Edit->Preferences->Better BibTex 中还有很多可以调整的选项。该插件的选项比较多，但在官方文档中对每一条选项都有较为详细的解释，需要深度定制的话建议通读官方文档。

Zotilo

Zotilo 提供了一系列 Zotero 实用功能的快捷实现方式，包括右键菜单和键盘快捷键两大类。通过适当的配置，Zotilo 可以大大提升 Zotero 的使用效率和操作舒适度，比如最实用的一个功能就是快速展开/隐藏界面左右侧的分组树和条目属性面板，从而增大中间条目区域的可视空间。

对该插件的详细介绍可参考这篇教程，这里直接贴出我的配置。

3 文献管理

导入数据

PDF 元数据检索

可以把下载好的 PDF 文件直接拖入 Zotero，拖入之后 Zotero 会自动检索并建立文献的元数据。（为了提高准确率，）Zotero 有一套比较繁琐的检索流程，因此在国内的网络环境下经常会检索特别慢或者干脆失败，可以通过挂载科学上网工具（全局模式）有效解决。桌面端软件本身没有网络设置的选项，目测是自动与浏览器的流量规则保持一致。

从剪贴板快速导入文献元数据

自动检索得到的元数据很多时候并不准确，这一点倒是无法避免，就算 Mendeley 也时不时会遇上这种情况。之前我都是手动把 BibTeX 格式的引用数据保存在本地，然后再在软件中选择从外部导入的，比较繁琐。现在 Zotero 可以直接从剪贴板导入数据，大大简化了这个步骤，只需要把文献数据直接复制到剪贴板，再到软件中按下 Ctrl+Shift+Alt+I 快捷键即可。

这个方法在导入中国知网的文献数据时特别有用，只要把 EndNote 格式的数据一键复制到剪贴板，就可以直接导入了，根本用不着很多教程讲的那么复杂。（所有 Zotero 支持导入的数据格式）

通过标识符添加文献元数据

直接输入一篇（或多篇）文献的标识符也可以快速导入数据，但这种方法同样对网络环境有要求。

另外由于个人不太喜欢通过浏览器插件的方式添加文献，故略过这部分内容。

校订数据

大小写切换

在英文中，对标题有两种书写方式，一种是标题大写 (Title case capitalization)，即标题中的主要单词首字母都大写；另一种是句首大写 (Sentence case capitalization)，即只有举首单词以及冒号/破折号后第一个单词的首字母大写。

Zotero 推荐使用句首大写，并提供了一个简单的转换器来切换这两种模式。在标题内容处右键鼠标即可弹出对应菜单。对于每一项条目的文献标题和收录标题（对于期刊文章来说就是期刊名，对于会议来说就是会议投递标题等等），都可以使用该转换器处理。

虽然但是，在实际使用中发现并不需要刻意关注这一项，因为即使是标题大写的形式，Zotero 一般也能正确处理。所以如果没有强迫症，大可以忽略掉。

姓名框切换

在条目的作者一栏中，有两种输入方式，默认是双输入框模式，还可以改为单输入框模式。双栏模式中，第一项是姓，后一项是名；单栏模式中不做区分，姓名按正常完整的顺序输入。实际使用中，中文汉字名称可考虑单栏，其余一般用双栏。

投递标题与会议名称

对于会议论文类型的条目，投递标题 (Proceedings Title) 和学术会议名称 (Conference Name) 是特别容易混淆的两个属性。前者指的是会议论文集，而后者指的是会议本身的名字。实际引用会议论文时，一般需要的是会议论文集的名称，即「投递标题」中的内容将被使用，而不是「学术会议名称」。在会议论文条目中，本应该划入投递标题的数据被导入到学术会议名称中，导致导出会议论文数据时生成的条目缺失会议项，是一个非常常见的问题，这一点要注意。如果遇到有的会议条目投递标题一栏空缺，而会议名称一栏有内容，应视内容将它们补充在投递标题一栏中。

Zotfile相关

ZotFile Rename

是 ZotFile 插件提供的功能，但属于文献的日常管理，所以放在这里讲。除了会将 PDF 标题修改为规定格式，该功能最常用来将文献迁移到自定义的目录下，也就是说它会把原本存在 Zotero 数据目录下的 PDF 文件转换为一个指向 OneDrive（对于我来说）中文件的软链接。

这也意味着导入一篇文献需要完成三步操作：拖入 PDF 文献，等待元数据检索完成并检查（或先导入元数据再拖入 PDF 文件），最后右键选择 Rename Attachments。比较繁琐，但没办法。

ZotFile Tablet

把文献发送到一个特定的文件夹，在编辑后能够取回并合并到主目录下，主要用于与便携式设备联动。我把这个 Tablet 文件夹设置成了 OneDrive 下另一个单独的文件夹，iPad 上使用 PDF Expert（免费版）等 APP 即可直接与这个 OneDrive 文件夹同步。

实际上这个功能有些鸡肋，iPad 既然能同步 OneDrive，直接访问 Zotero 的主文件目录就好了，何必画蛇添足，所以我考虑这个功能主要是面向文献库主目录不在 OneDrive 上的情况（文献目录在电脑端本地或其它不支持的网盘），或者文献量太大、直接在主文件目录中不方便查找的情况。但不管哪种情况，要使用该功能都需要在电脑上提前操作，手动把文献发送到 Tablet。所以目前我没有用到很多这个功能。

空文件夹删除

ZotFile 还有一个很大的缺陷就是在删除文件的时候无法一并删除它在文件目录下创建的文件夹，有两个办法善后。

借助 Everything 软件。Everything 的高级语法支持查询空文件夹（empty: ）：

这个方法首先需要安装 Everything，其次不能识别“文件夹套空文件夹”的情况。

  1. 参考这个教程，写一个 dos 批处理文件，可以直接复制下面的代码到一个文本文件，把后缀改成 .bat，拖到文献目录，以后每次清理直接双击运行就好了：

@echo off
setlocal enabledelayedexpansion
del /q c:\empty_dir
del /q c:\directory.txt
REM 下面的代码加上sort /r ，表示逆序排列文件夹。子文件夹在前，父文件夹在后。
dir /a:d /b /s “.” | sort /r > “%cd%\directory.txt”
for /f “usebackq delims=” %%i in (“directory.txt”) do (
rem echo “enter dir is: %%i”
rem cd “%%i”
rem echo “the bat path is: %~f0” rem print this batfile’s whole path.
echo “cur dir is: “%%i””
REM 将当前目录下的所有文件打印到以下文件中：
dir /a /b “%%i” >”c:\folder_content.txt”
rem echo “=======================================================”
rem echo “==================fold content: begin =================”
:type “c:\folder_content.txt”
rem echo “==================fold content: end =================”
rem echo “=======================================================”

REM 下面的findstr命令查找当前文件是否有内容，如果查找成功表示目录不是空的，否则是空的。
REM 注意下面的两个 '与' 和 两个 '或' 符号。    
findstr . "c:\folder_content.txt" >nul 2>nul && ( echo "The folder "%%i" is NOT NOT NOT empty") || ( echo "The folder "%%i" is empty" && echo "%%i">>"%cd%\empty_dir" && rd "%%i")
del "c:\folder_content.txt"
)

del “%cd%\directory.txt”
标签

可以给文献打标签，标签还可以指派颜色，最多指派 9 种颜色（也就是最多 9 个标签可加颜色）。指派颜色的标签会在对应文献标题处显示一个小色块，辨识度很高。不指派颜色的标签在选中对应文献所在分组之后才会显示在标签选择器中，而指派颜色后的标签在所有分组下都会显示在标签选择器中。指派颜色的标签，还可以直接通过键盘数字键 1-9 按对应的顺序快速分配给选中的文献。

标签的分类维度应该与文献本身的分组区分开来，可以考虑应用维度或者重要性维度。

Zotero 自带了部分快捷键：

可以看到，自带的快捷键数量非常有限，但如前所述，可使用 Zotilo 插件大大拓展快捷键的种类，从而加强键盘的操作，不再赘述。

除上面提到的快捷键外，还有一些快捷操作值得注意：

查看某一分组下的条目总数，点击该分组即可；或先单击某一条目再按下 “Ctrl+A”（全选）。

  1. 右侧图标目录下可以快速打开不同搜索引擎对某篇文献的检索结果。目前无法通过 GUI 添加新的检索引擎，需要直接修改配置文件，可参考这个讨论和这个教程。

  2. 指派颜色的标签，可以通过键盘数字键 1-9 按对应的顺序快速分配给选中的文献（前面已提到过一次）。

5 文献引用

引文格式

Zotero 支持 Citation Style Language (CSL)，借助 CSL 项目的数据库，大多数正规的英文期刊都有现成的引文格式可供使用，不需要自己重新编写。

在 Zotero 设置中可以直接搜索对应期刊的引文格式，点击按钮 ① 输入期刊名称即可。Tip: 如果搜不到期刊的名字，可以试试该期刊所属的出版商/机构名称，会有惊喜。
对于离线环境，或者其它定制的 CSL 样式（一般是别人写好的学校毕业论文样式），可以通过按钮 ② 导入样式文件。
默认经典视图

Zotero 在 Word 里默认的引文添加视图是一个搜索框。其实我很反感这种搜索框，谁还能在一股脑导入二三十篇文献的时候记得每一篇该搜什么关键词？还是经典视图比较舒服，点点鼠标，一目了然。还好 Zotero 可以修改默认的引文选择视图，省得每次都多点一步：

同一位置批量添加

与 Mendeley 不同，如果在一个位置多次添加引文，并不会得到多个连续的引用标志（就类似先插入 [1] 再在后面插入 [2]，最后得到 [1][2]），而只会在原有的基础上调整（[1] 被替换为 [2]，甚至 [1] 数字不变，但对应的引文变成了原本 [2] 的引文）。如果需要在同一位置插入多个文献，记得选择“多重来源”，否则会将原有的文献挤掉。

另外，对于插入 [1] 后，加个空格再插入 [2] 的方式，虽然也可以得到分开的 [1] 和 [2]，但其本质上只是用空格隔开了两个单独的域，如果多重引用的样式发生改变（如规定为 [1, 2] 或 [1-2]），这两个割裂的域并不能随之自动合并调整，所以不推荐这么做。

6 缺陷

在使用中发现 Zotero 也不是一个完美的文献管理方案，以下是一些不好的体验。

笔记视图不能展开预览

Mendeley 可以在右侧的属性框快速查看一个条目下的完整笔记，但是 Zotero 中笔记的预览视图却被标签化，只能显示第一行的内容，要完整查看必须点进去。多了一层入口，打断了浏览状态，反而降低了效率。诚然 Zotero 可以添加多条笔记，这是 Mendeley 所没有的功能，但个人感觉很鸡肋，倒不如一眼扫过去尽收眼底舒服。（与我对笔记的使用方式也有关，大部分情况下是三两句话的简单信息，不会拿它做大段篇幅的阅读笔记）

不支持全文高亮搜索

这一点算是 Mendeley 的强项了，恰恰也是 Zotero 最薄弱的环节。由于没有内置阅读器（及其它种种原因），Zotero 可以对 PDF 全文搜索，但是不能把搜索结果高亮展示出来。这不是一个新问题了，而且在可见的未来也不太可能得到完善。

对于这个问题，可以搭配使用 [支持特定路径下 PDF 全文搜索的 PDF 阅读器] 来做一定程度的补偿，如 PDF-XChange Viewer。

移动设备无官方 APP

倒是有一款 PaperShip，不过很可惜不能同步 OneDrive（但是可以连坚果云，因为支持 WebDAV）。另外，PaperShip 的免费版没有编辑权限，想编辑？得加钱！（雾

PDF Expert 是我目前使用的方案，免费版可以连接到 OneDrive，也能够编辑，唯一的不足就在于不能按照文献分组的树形结构去浏览（所以前面我提到按照 %c 建立存储目录）。

引文添加流程繁琐

插入多个引文时需要显式地声明是“多重来源”就很弱智，为什么不能按着 Ctrl 或 Shift 直接选中多个文献呢？至少，可以做到检测到选中多个文献就自动转为多重来源视图吧。

配置繁琐，对小白不友好？

这也是跟 Mendeley 截然相反的一点。不过我觉得见仁见智，至少我倒是乐在其中的（穷折腾实锤了）。而且配置灵活固然带来相对繁琐的初期操作，但是也在一定程度上提高了软件的自由度。综合考虑，把这一点放在最后提一下。
